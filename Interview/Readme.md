#### 🗣️ 리액트 CSR의 개념에 대해 설명해주세요

```
CSR은 보통 SPA(Single Page Application)에서 쓰이는 기법으로,
리액트가 렌더링 될 때, 로컬호스트같은 docment(index.html > index.js(포함))
⇒ style ⇒ script 순으로 코드를 읽어오는 방식을 의미합니다.
이 때 document에는 HTML 인덱스파일과 메타 데이터가 존재하는데
기본은 텅 비어져 있습니다.
따라서 아무것도 보여주지 않다가,
사용자의 기기에서 렌더링이 진행되어 한 번에 화면에 전달해주는 방식입니다.

**요약** :  index.html을 다운 받게 되고 그 안의 index.js 파일을 다운로드하고
리액트 라이브러리까지 다운 받은 후 페이지로딩을 하는 방식을 의미
```

Next.js SSR의 특징에 대해 설명해주세요


SSR의 방식의 단점은 무엇인가요?

```
1. 매 요청마다 서버에서 렌더링 작업을 하기 때문에 트래픽이 많이 발생하는 경우 서버 과부화가 걸리기 쉽습니다.
2.  초기 로딩 시간이 CSR방식에 비해 오래걸립니다.
    CSR은 초기에 비어있는 HTML을 전송하고, 이후에 페이지를 구성하지만
    SSR은 클라이언트에서 완전히 렌더링 페이지를 제공하기 때문입니다.
3.  서버에서 렌더링을 하다보니 웹 페이지를 서버에서 받아오는 것과
    동일하기 때문에 Blinking(깜빡거림현상)이 발생되어 UX에 좋지 않을 수 있습니다.

**혹시나 모르니까.. 해결 방법은 skeleton 화면을 사용하여
초기 로딩 시에도 페이지구조와 스타일을 미리 표기
```

#### 🗣클래스형 컴포넌트와 함수형 컴포넌트의 차이점에 대해 설명해주세요

```

class 키워드를 사용하고 React.component를 상속받습니다.

함수 내부에서 JSX를 반환합니다.
클래스형 컴포넌트에 비해 간단하고 직관적인 문법을 사용합니다.

함수형 컴포넌트에서는 useEffect()를 사용해서 라이프사이클 이벤트에
 대응합니다.
```

#### 브라우저 렌더링 원리에 대해 설명해보세요

```
브라우저 렌더링은 브라우저가 화면에 그리는 과정을 의미합니다.
렌더링 순서는 CriticalRendering Path에 따릅니다.
먼저 HTML을 파싱하여 DOM트리를 생성하고 ⇒
DOM트리는 HTML 태그를 노드로 변환해서 노드간 계층 관계를 형성합니다.
⇒ CSS파싱하여 스타일정보를 분석하여 CSSOM을 형성합니다
⇒ DOMTree + CSSOM이 결합되어 “RenderTree”를 생성합니다
⇒ 이후 레이아웃을 통해 배치가 이뤄지고,
페인트를 통해 화면에 스타일이 그려져 이것이 합쳐져(합성)서
최종적으로 스크린에 나타나게 됩니다.
```

#### 클로저란 무엇인가요?

```
자바스크립트 클로저(Closure)는 함수와 그 함수가 선언된 렉시컬 환경
(Lexical Environment)의 조합입니다. 클로저(innerFunc)는
함수가 자신의 스코프 외부에서 정의된 변수에 접근할 수 있는
능력을 의미합니다. 즉, 함수가 선언될 당시의 환경을 기억하고,
이후에도 해당 환경에 접근할 수 있는 것입니다.
```

```
호이스팅은 자바스크립트에서 변수와 함수 선언을
해당 스코프의 최상위로 옮기는 동작을 의미합니다.
```

호이스팅과 관련하여 변수 var, let, const에 대해 차이점을 설명해보세요

```
var는 함수 스코프 변수(함수 내부) 또는 전역 스코프(함수 외부)로
 호이스팅이 일어나며 중복 선언과 재할당이 가능합니다.
 let과 const는 블록 스코프 변수로 호이스팅이 일어나지만
 TDZ(Temporal Dead Zone) 개념이 도입되어
 선언부 이전에 변수에 접근할 수 없습니다.
 let은 중복 선언이 금지되고 재할당이 가능하며,
 const는 중복 선언과 재할당이 모두 금지됩니다.
```

#### 변수의 생성 단계에 대해 설명해보세요

```
- 1단계는 선언 단계입니다. 선언 단계에서는 변수를 실행 컨텍스트의 변수 객체에 등록하게 되는데 이 변수 객체는 스코프가 참조하는 대상이 됩니다.
- 2단계는 초기화 단계입니다. 초기화 단계에서는 변수 객체에 등록된 변수를 위한 메모리 공간을 확보합니다. 이 때, 변수는 undefined로 초기화됩니다.
- 3단계는 할당 단계입니다. 할당 단계에서는 undefined의 값에 실제 value가 할당됩니다.
```

#### ==와 ===의 차이는 무엇인가요?

```
"=="는 동등 연산자로, 두 값을 비교할 때 형변환(type coercion)을
 수행합니다. 즉, 비교하는 값의 데이터 타입이 다르더라도
 자동으로 형변환을 수행한 후에 비교합니다.
 이러한 형변환은 때로 예측하지 못한 결과를 초래할 수 있으므로,
 "=="를 사용할 때는 조심해야 합니다.

 "==="는 일치 연산자로, 두 값이 데이터 타입과 값이 모두 같은지 비교합니다.
 따라서 "==="를 사용하면 형변환 없이 정확한 값을 비교할 수 있습니다. 이러한 일치 연산자를 사용하는 것이 더 안전하고 예측 가능한 결과를 얻을 수 있습니다.
```

```
Cors(Cross-Origin-Resource Sharing)는 스크립트의 다른
 출처로부터 리소스를 요청할 때 발생하는 보안 정책입니다.
 예를 들어 Client와 Server간 통신할 때
 Cors 이슈가 발생할 수 있습니다.

작업 하면서 여러 번 CORS 이슈에 대해 경험한 적이 있습니다.
서버에 대한 개념을 모를 때 API를 불러와서 통신할 때 겪어본 적이 있습니다.
CORS를 처리하기 위해서 서버 측에서 Client와의 도메인을 일치시켜서 해결한 적이 있습니다.
```

CORS 라이브러리 설치 말고 다른 해결 방법은 없나요?

```
그 외 프록시를 사용해서 CORS 이슈를 해결할 수도 있습니다.
프록시는 클라이언트와 서버 사이에 요청을 중계하는 중간 서버 역할을 합니다.
```

#### 브라우저캐싱, 네트워크캐싱에 대해 개념을 설명해주세요

```
네트워크 캐싱은 웹 리소스의 임시 저장소인 캐시를 활용하여 리소스의 다운로드와 전송을 최적화하는 기술입니다.

- **브라우저 캐싱**(로컬에 저장): 웹 페이지의 리소스를 브라우저 내부에 저장해두고,
같은 리소스를 요청할 때 서버에 요청하지 않고 캐시에서 불러옵니다.
이를 통해 웹 페이지의 로딩 속도를 향상시키고 네트워크 트래픽을 줄일 수 있습니다.
- **네트워크 캐싱**(서버-클라이언트 중간서버에 저장): 리소스가 서버에서
 브라우저로 전송될 때,
 브라우저나 프록시 서버에 해당 리소스를 저장해두고 다음에 같은 리소스가
  요청될 때 다시 서버로부터 받지 않고 캐시에서 불러옵니다.
  이를 통해 네트워크 대역폭을 절약하고 서버의 부하를 줄일 수 있습니다.
```

**📝[헷갈리면 이것만 알아두자]**

차이점은 Cache 저장 위치다.

브라우저 캐싱은 로컬에, 네트워크 캐싱은 프록시 같은 중간서버에 리소스를 저장해둔다. 하지만 모두 리소스 재사용을 통해 웹 성능을 향상시키고 네트워크 트래픽을 줄이는 데 도움을 주기 위해 사용된다.

#### 네트워크에서 CDN은 무엇인가요? 개념과 특징에 대해 설명해주세요

```
CDN은 “Content Delivery Network”의 약자로, 콘텐츠 전송
 네트워크를 의미합니다. CDN은 전세계에 분산된 서버로 구성되어 있으며,
  CDN은 주로 웹사이트나 애플리케이션의 정적 콘텐츠
  (이미지, CSS, JavaScript 파일 등)을 빠르게 제공하기 위해 사용됩니다.

  1. **캐싱과 가까운 서버**: CDN은 콘텐츠를 전세계 여러 위치에
   캐싱하여 사용자에게 가까운 서버에서 콘텐츠를 제공합니다.
   이를 통해 지리적으로 먼 사용자도 빠른 속도로 콘텐츠를 받을 수 있습니다.

2. **로드 밸런싱**: CDN은 트래픽을 여러 서버로 분산시켜 로드 밸런싱을 수행합니다.
이를 통해 단일 서버에 부하가 집중되지 않고, 안정적인 서비스를 제공할 수 있습니다.
3. **보안 및 안정성**: CDN은 DDoS 공격 등과 같은 네트워크 공격을
 탐지하고 막는 등의 보안 기능을 제공합니다.
 또한, 콘텐츠 전송 중에 발생할 수 있는 네트워크 장애를 자동으로 처리하여 안정성을 향상시킵니다.
4. **비용 및 성능 개선**: CDN을 사용함으로써 웹사이트나 애플리케이션의
 성능을 개선하고, 트래픽을 효율적으로 처리할 수 있습니다.
 또한, 웹 호스팅 비용을 절감할 수 있습니다.
```
